Apache Atlas

Author: Damon WW Date: July 13, 2020  15:44:04

Apache Atlas是什么

Atlas是Hadoop的数据治理和元数据框架。

Atlas是一组可扩展和可扩展的核心基础治理服务，使企业能够有效，高效地满足Hadoop中的合规性要求，并允许与整个企业数据生态系统集成。

Apache Atlas为组织提供了开放的元数据管理和治理功能，以建立其数据资产的目录，对这些资产进行分类和治理，并为数据科学家，分析师和数据治理团队提供围绕这些数据资产的协作功能。

Apache Atlas架构

Atlas包括以下组件：

  • 采用Hbase存储元数据
  • 采用Solr实现索引
  • Ingest/Export 采集导出组件 Type System类型系统 Graph Engine图形引擎 共同构成Atlas的核心机制
  • 所有功能通过API向用户提供，也可以通过Kafka消息系统进行集成
  • Atlas支持各种源获取元数据：Hive，Sqoop，Storm，hbase，falcon等
  • 还有UI支持

功能

  • Atlas支持各种Hadoop和非Hadoop元数据类型
  • 提供了REST API进行集成
  • 对数据血缘的追溯达到了字段级别
  • 支持对权限的控制

核心概念

  • Types
    Atlas中的 “类型” 定义了如何存储和访问特定类型的元数据对象。类型表示了所定义元数据对象的一个或多个属性集合。可以将 “类型” 理解成面向对象的编程语言的 “类” 定义的或关系数据库的 “表模式”。atlas里内置了支持的数据源的相关元数据Type，如果需要新增新的Type可以通过API来创建。

与 Atlas 本地定义的类型的示例是 Hive 表。 Hive 表用这些属性定义：

  • Entities
    可以理解为“类”的实例，相当于Type的一条实例记录。

  • Hook
    各个数据源的元数据的接入都是通过hook的方式进行了，通过hook，atlas可以实时侦听数据源上的相关操作，通过解析对应的操作，获取元数据和血缘关系等信息。

如hive hook会hook以下的操作：

  • create database
  • create table/view, create table as select
  • load, import, export
  • DMLs (insert)
  • alter database
  • alter table (skewed table information, stored as, protection is not supported)
  • alter view

通过hook这些操作，基本上新建表，修改表引起的元数据的新增和修改都会导入到atlas中，load、import、export的数据导入导出操作，则可以获取到相关的数据输入和输出路径，DMLs的操作则可以获取到字段的血缘关系

除了atlas默认支持的数据源外，其他的数据源的接入就需要我们自己开发相应的代码hook数据源的相关操作，来解析元数据和相关的血缘关系

Atlas的优缺点

  • 优点：
  1. 提供了rest api和messaging两种接入元数据方式，相对应地，我们可以

通过hook的方式实时接入数据源的相关元数据信息，相当于使用atlas来对元数据的收集和管理
通过API的方式输入我们自己整理好的元数据信息，相当于只使用atlas来对我们的元数据信息进行管理
2. 架构设计比较容易扩展

  1. 提供的UI界面比较优秀

  2. 官方文档较为丰富

  • 缺点：

  1. 提供的数据源比较有限，sql相关的数据源，如mysql、oracle需要我们自己开发相关的hook组件进行接入

  2. github星比较少，相关资料比较少
